# CLAUDE.md

## 專案目的

本專案用於研究與分析兩大類市場相關人物：

**A. 市場觀察方（investment-institutions/）**
涵蓋**策略師、分析師、知名投資人**等發表市場看法的人物。透過系統性地記錄與追蹤這些人過去公開發表的言論、預測與立場，目的是：

1. 理解每個人的**歷史預測紀錄**（預測 vs. 實際結果）
2. 分析其**長期立場變化**（多頭/空頭/中立，以及轉向的時機與理由）
3. 歸納其**人格特質與思考框架**（例如：偏好逆向操作、重視總經、技術派、價值派等）
4. 評估**哪些人的市場看法值得參考**

**B. 公司經營方（public-companies/）**
涵蓋**上市公司 CEO、創辦人、CFO、關鍵高管**等經營層人物的專訪與公開發言。目的是：

1. 追蹤其對**自家公司營運、產業趨勢、競爭格局**的公開論述
2. 比對其**過去承諾與實際執行結果**（財測、產品時程、併購、資本配置等）
3. 觀察其**溝通風格與人格特質**（保守 vs. 樂觀、透明 vs. 模糊、是否有過度承諾傾向）
4. **重要差異**：CEO/高管對自家公司必有**利益衝突與立場偏誤**，其發言需視為「帶立場的一手資訊」，而非中立觀察

## 語言規範

**本專案所有的整理、回覆、檔案內容一律使用「繁體中文」**。

- 與使用者對話時使用繁體中文
- 建立或更新的所有 Markdown 檔案（Summary.md、個別發言紀錄等）內容使用繁體中文
- 專有名詞、人名、機構名稱可保留原文（例如：Goldman Sachs、Jerome Powell）
- 引用的原始發言可保留英文原文，但需附上繁體中文摘要或翻譯

## 工作流程指引

### 判斷人物歸屬（優先步驟）

收到新人物時，先判斷應歸入哪個資料夾：

- 若該人物身分為**發表市場看法的觀察者**（賣方分析師、策略師、基金經理、獨立評論者）→ `investment-institutions/`
- 若該人物身分為**上市公司的經營者或關鍵高管**（CEO、創辦人、CFO、技術長、董事長）→ `public-companies/`
- 若身分重疊（例：避險基金經理人同時也是公開上市公司大股東兼董事），則依本次發言內容的**主要身分**歸屬，並在 Summary.md 備註交叉關係

### 當使用者要求**新增一位人物**時

1. 確認應歸屬 A 或 B 類
2. 確認機構/公司名稱與人名（英文原名）
3. 建立對應資料夾：`investment-institutions/{機構}/{人名}/` 或 `public-companies/{公司}/{人名}/`
4. 依照對應格式建立 `Summary.md` 並先填入已知的基本資料
5. **若為 B 類（CEO/高管），必須填寫「利益衝突揭露」章節**，即使初期資訊有限也要先列出通則
6. 回報使用者目前填入的內容與尚待補充的欄位

### 當使用者提供**一則新的發言或 URL** 時

1. 若人物尚未建檔，先建立該人物資料夾與 `Summary.md`
2. 將發言加入 `Summary.md` 的索引表格（A 類為「歷史發言索引」、B 類為「歷史專訪/發言索引」）
3. 若發言內容豐富或具代表性，另外建立 `{日期}-{主題}.md` 詳細檔案
4. **若為 B 類**：從該場發言中擷取「具體承諾 / 預測」加入 Summary.md 的「承諾追蹤紀錄」表格（待追蹤狀態）
5. 視情況更新「主要立場與風格」/「主要敘事與核心論述」與「人格特質觀察」/「溝通風格」章節

### 當使用者要求**分析某人物**時

1. 讀取該人物的 `Summary.md` 與相關個別檔案
2. 依據歷史紀錄提供客觀分析，避免加入未經佐證的推測
3. **若為 B 類（CEO/高管）**：分析時必須明確區分「關於自家公司的發言（帶立場）」與「關於產業/競爭對手的發言（仍可能帶立場但較中立）」，不可混為一談
4. 若資料不足，明確告知使用者哪些資訊缺失

## 命名規範

- **機構資料夾**（A 類）：使用英文原名，空格以 `-` 取代（例如：`Goldman-Sachs`）
- **公司資料夾**（B 類）：優先使用 Ticker（例如：`NVDA`、`TSLA`、`AAPL`）；若為台股或非美股且 Ticker 不易辨識，可用英文公司名（例如：`TSMC`、`Samsung-Electronics`）
- **人名資料夾**：使用英文原名，空格以 `-` 取代（例如：`David-Kostin`、`Jensen-Huang`）
- **個別發言檔案**：`{YYYY-MM-DD}-{簡短主題}.md`，主題可使用繁體中文

## 客觀性原則

- 記錄發言時應**忠實呈現原意**，避免加入主觀評論
- 「預測追蹤」/「承諾追蹤」欄位應以**事實佐證**，標註實際結果的資料來源
- 「人格特質觀察」/「溝通風格」應基於**多次發言的歸納**，而非單一事件的印象
- 如引用他人對該人物的評價，需註明評價來源

### B 類（公司經營方）特別原則

- **立場偏誤警示**：CEO/高管對自家公司的正面論述屬於**預設帶立場**，不可當作客觀事實記錄；應以「{人名}表示 / 宣稱 / 預期」等措辭呈現，而非「該公司將...」
- **承諾 vs. 願景**區分：
  - **承諾**：具時程、可驗證（例：「2025 年底前推出 X 產品」、「明年毛利率達 50%」）→ 必須進入承諾追蹤表
  - **願景**：無明確時程或指標（例：「我們會繼續創新」）→ 僅記於敘事章節，不進入追蹤表
- **跨公司評論需特別謹慎**：CEO 對競爭對手、供應商、客戶的評論可能涉及商業利益衝突（例：施壓供應商、貶低對手產品），需在記錄時特別標註該段發言的潛在動機

## 來源與可信度要求（強制）

本專案的研究價值完全建立在**資料來源的可追溯性與可信度**之上。任何分析、發言紀錄、預測追蹤、人格特質歸納，都必須滿足下列要求：

### 1. 必須標註參考 URL

- **任何一筆發言紀錄都必須附上至少一個參考 URL**，無論是記錄在 `Summary.md` 的「歷史發言索引」或是個別發言詳細檔案
- 若原始文件為非公開資料（例：券商內部 desk note、付費終端），仍須註明：
  - 該資訊的**轉述來源 URL**（中/英文媒體報導、社群轉貼等）
  - 原始文件的**性質與取得管道**（例：「高盛內部 desk note，非公開」）
- 連結格式使用 Markdown：`[來源名稱](URL)`

### 2. 必須驗證可信度

在將一則發言納入紀錄**之前**，必須先進行可信度檢核：

- **第一手 vs 第二手**：優先使用第一手來源（發言者本人/所屬機構官方發布、知名通訊社完整引用）；若僅有第二手轉述，須註明
- **交叉驗證**：重要論點（具體數字、預測、立場轉向）應**至少有兩個獨立來源**相互佐證
- **來源分級**：在來源連結旁可標註分級，例如：
  - ✅ **一手/高可信**：發言者官方、彭博/路透/Bloomberg/Reuters/WSJ/FT 等一線財經媒體、機構官方 PDF
  - ◯ **二手/中可信**：中文財經媒體、產業部落格、X/Twitter 帳號（需進一步驗證）
  - ⚠️ **未驗證/低可信**：論壇、匿名爆料、單一來源的 AI 摘要
- **時間敏感資訊**（例：發言日期、價格目標）要特別留意是否為原始報導，避免日期被轉述過程中錯置

### 3. 謠言、未驗證資訊的特別註解

若資訊無法通過可信度驗證，仍可記錄，但**必須顯著標註**：

- 使用 `⚠️ 未經證實` 或 `⚠️ 來源存疑` 作為開頭標記
- 註明**為何無法驗證**（例：找不到原始出處、僅有單一中文轉述、來源為匿名社群）
- **禁止**將未驗證資訊直接寫入 `Summary.md` 的主論述、主要立場、預測追蹤欄位
- 未驗證資訊建議放在個別發言檔案的**「備註」或「待驗證」**章節，或在 `Summary.md` 中以獨立區塊呈現

範例：

```markdown
## ⚠️ 待驗證資訊
```

### 4. 使用者提供資訊的處理原則

- 若使用者直接提供一段文字（原始報告摘要、他人整理稿），**應視為「待驗證的二手資訊」**，不可直接當作一手來源引用
- 應主動以 WebSearch / WebFetch 查證關鍵事實（人名、職稱、機構、日期、數字、原始標題）
- 若查證結果與使用者提供內容不一致，**應明確向使用者回報差異**，由使用者裁決最終版本
- 若使用者明確指出某項事實（例：發布日期），使用者本身即為**權威來源**，可據以覆蓋其他二手資料，但需在備註中留下更新紀錄

### 5. 定期檢視與降級

- 過去被記錄為「已驗證」的資訊，若後續發現原始來源失效（404、付費牆、媒體下架），應降級並標註
- 若某來源多次被發現有錯誤轉述紀錄，應在 `Summary.md` 備註中標註該來源的可信度問題
