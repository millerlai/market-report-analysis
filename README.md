# Market Report Analysis

追蹤與分析市場相關人物的公開發言、預測與立場，建立可長期回顧、可事後驗證的個人化研究資料庫。

本專案採用結構化的 Markdown 檔案組織人物資料，並搭配 [Claude Code](https://docs.claude.com/claude-code) 的 `CLAUDE.md` 規範，讓 AI 助手能依照一致格式建檔、更新、追蹤預測結果。

---

## 專案目的

分為兩大類追蹤對象：

### A. 市場觀察方 `investment-institutions/`
賣方分析師、策略師、基金經理、知名投資人等**發表市場看法的觀察者**。

目的：
1. 建立**歷史預測紀錄**（預測 vs. 實際結果）
2. 追蹤**長期立場變化**（多頭／空頭／中立的轉向時機與理由）
3. 歸納**人格特質與分析框架**（總經派、基本面派、技術派、逆向派等）
4. 評估**哪些人的市場看法值得參考**

### B. 公司經營方 `public-companies/`
上市公司 CEO、創辦人、CFO 等**經營層人物**的專訪與公開發言。

目的：
1. 追蹤對**自家公司營運、產業趨勢、競爭格局**的公開論述
2. 比對**過去承諾與實際執行結果**（財測、產品時程、併購、資本配置）
3. 觀察**溝通風格與人格特質**（保守 vs. 樂觀、是否過度承諾）
4. **識別立場偏誤**：CEO 對自家公司必有利益衝突，發言視為「帶立場的一手資訊」而非中立觀察

## 資料夾結構

```
investment-institutions/      # A. 市場觀察方
├── {機構名稱}/                # 例：Goldman-Sachs
│   └── {人名}/                # 例：David-Kostin
│       ├── Summary.md          # 人物總覽與歷史發言索引
│       └── {YYYY-MM-DD}-{主題}.md    # 個別發言詳細紀錄

public-companies/             # B. 公司經營方
├── {公司 Ticker}/             # 例：NVDA
│   └── {人名}/                # 例：Jensen-Huang
│       ├── Summary.md          # 含利益衝突揭露、承諾追蹤表
│       └── {YYYY-MM-DD}-{主題}.md    # 個別專訪詳細紀錄
```

命名規範詳見 `CLAUDE.md`。

## 如何使用（搭配 Claude Code）

### 1. 新增一位人物

直接告訴 AI 助手：

> 幫我新增高盛的首席美股策略師 David Kostin

或針對 CEO：

> 請把 Tesla 的 Elon Musk 加進來

助手會：
- 判斷應歸屬 A 類或 B 類
- 建立對應資料夾與 `Summary.md`
- 填入已知基本資料，並回報尚待補充的欄位

### 2. 記錄一則發言或專訪

提供 URL 即可：

> 整理這篇專訪：https://...

助手會：
- 驗證一手 vs. 二手來源（優先引用原始報導、影片、逐字稿）
- 若需要會以 WebSearch／WebFetch 交叉驗證日期、數字、人名
- 將發言加入 Summary.md 索引表
- 若內容豐富，另建立 `{日期}-{主題}.md` 詳細檔
- **B 類**：額外擷取「具體承諾」加入承諾追蹤表

### 3. 分析某人物

> 幫我分析 Jensen Huang 的公開發言立場

助手會依據 `Summary.md` 與個別檔案提供客觀分析；若資料不足，會明確回報哪些欄位缺失。

## 客觀性與可信度要求（強制）

本專案的研究價值建立在**資料來源的可追溯性**之上。所有發言紀錄須遵守：

1. **必須標註參考 URL**：任何一筆發言至少附一個來源連結
2. **來源分級**：
   - ✅ **一手／高可信**：發言者本人／機構官方、Bloomberg／Reuters／WSJ／FT 等一線財經媒體、機構官方 PDF
   - ◯ **二手／中可信**：中文財經媒體、產業部落格、待驗證社群帳號
   - ⚠️ **未驗證／低可信**：論壇、匿名爆料、單一 AI 摘要
3. **交叉驗證**：重要數字、預測、立場轉向應至少有兩個獨立來源
4. **謠言不進主論述**：未驗證資訊須以 `⚠️ 待驗證` 區塊獨立呈現，禁止混入主要立場、預測追蹤欄位
5. **B 類特別原則**：
   - CEO 對自家公司的正面論述**預設帶立場**，以「{人名}表示／宣稱／預期」措辭呈現
   - 區分**承諾**（具時程、可驗證）vs. **願景**（無指標），僅前者進入承諾追蹤表
   - 跨公司評論（對手、客戶、供應商）須標註潛在商業動機

完整規範見 [`CLAUDE.md`](./CLAUDE.md)。

## 語言規範

- 所有檔案內容、AI 回覆一律使用**繁體中文**
- 人名、機構名、公司名可保留原文（例：Goldman Sachs、Jensen Huang）
- 引用的原始英文發言可保留原文，另附繁體中文摘要或翻譯
